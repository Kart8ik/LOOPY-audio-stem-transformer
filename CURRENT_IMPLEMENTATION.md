# CURRENT_IMPLEMENTATION

## 1) What this project is

LOOPY is a monorepo with:
- A React + Vite frontend (workspace: frontend)
- A FastAPI + Demucs backend (workspace: backend)

Main user flow:
1. User uploads mp3/wav in frontend.
2. Frontend sends file to backend endpoint POST /upload-and-process.
3. Backend runs Demucs vocal separation and exposes instrumental as static file.
4. Frontend lets user select a region and desired duration.
5. Frontend sends region + duration to backend endpoint POST /loop.
6. Backend builds looped mp3 and returns it as downloadable file.

## Backend workspace (backend)

### backend/package.json
Purpose:
- Backend workspace metadata and helper scripts.

Scripts:
- dev: uvicorn with reload on port 3000
- start: uvicorn without reload on port 3000
- install: pip install -r requirements.txt

### backend/requirements.txt
Purpose:
- Python dependency pin set.

Core groups:
- API stack: fastapi, starlette, uvicorn, pydantic
- Audio/ML stack: demucs, torch, torchaudio, openunmix, julius, pydub
- Utility and support libraries required by Demucs/PyTorch ecosystem

Important runtime note:
- python-multipart must be installed for file upload endpoints using UploadFile.
- If missing, FastAPI raises runtime error during app import.

### backend/server.py
Purpose:
- FastAPI app creation, middleware setup, route definitions, static file mounting.

Global setup:
- Enables CORS for localhost origins.
- Ensures temp_uploads and separated directories exist.
- Mounts /processed static route to backend/separated.

Data model:
- LoopRequest with filepath, startTime, endTime, loopDuration.

Endpoints:
- GET /trial
  - Health/welcome test endpoint.

- POST /upload-and-process
  - Accepts mp3/wav UploadFile.
  - Cleans previous temp and processed output directories.
  - Writes upload to temp_uploads with UUID filename.
  - Calls separate_vocals(file_path).
  - Computes expected output path separated/mdx_extra/<uuid>/no_vocals.mp3.
  - Returns processed_url and processed_filepath.

- POST /loop
  - Accepts LoopRequest JSON.
  - Calls create_loop(...) from loopy.py.
  - Returns mp3 file response for download.

### backend/loopy.py
Purpose:
- Pure processing layer for stem separation and looping.

Functions:
- separate_vocals(input_file, output_dir='separated', model='mdx_extra')
  - Builds demucs CLI command with --two-stems vocals.
  - Exports no-vocal stem as mp3 into output dir.

- create_loop(filepath, start_time, end_time, loop_duration_minutes)
  - Loads audio with pydub.
  - Slices selected segment by start/end times.
  - Validates segment is non-empty.
  - Repeats segment until requested duration.
  - Applies small fade-in to subsequent repetitions for smoother joins.
  - Exports to separated/looped/<uuid>.mp3 and returns path.

## 3) End-to-end runtime sequence

## Startup sequence
1. Root command npm run dev starts frontend and backend in parallel.
2. Frontend runs on localhost:5173 (Vite).
3. Backend runs on localhost:3000 (Uvicorn + FastAPI).

## Upload + process sequence
1. User uploads file in Loopy page.
2. Frontend sends multipart/form-data to backend /upload-and-process.
3. Backend validates extension, stores file, runs Demucs.
4. Backend responds with static URL and absolute/relative processed filepath.
5. Frontend loads processed URL into waveform UI.

## Loop creation sequence
1. User sets region and duration.
2. Frontend sends JSON payload to backend /loop.
3. Backend slices and repeats selected section in loopy.py.
4. Backend streams resulting mp3 as FileResponse.
5. Frontend creates blob URL and enables download.

## 5) Operational checklist

For local development:
1. Root: npm install
2. Backend: create and activate backend/.venv
3. Backend: pip install -r requirements.txt
4. Run both: npm run dev
5. Test backend health: GET http://localhost:3000/trial
6. Test UI: open http://localhost:5173
